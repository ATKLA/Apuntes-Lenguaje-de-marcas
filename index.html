DTD y XSD: Definición de Esquemas XML

1. Ejemplo de DTD (Document Type Definition)

<!DOCTYPE biblioteca [
  <!ELEMENT biblioteca (libro+)>
  <!ELEMENT libro (titulo, autor, año, precio)>
  <!ELEMENT titulo (#PCDATA)>
  <!ELEMENT autor (#PCDATA)>
  <!ELEMENT año (#PCDATA)>
  <!ELEMENT precio (#PCDATA)>
  <!ATTLIST libro idioma CDATA #IMPLIED>
]>

Puntos clave de DTD:

  • Sintaxis simple basada en declaraciones <!ELEMENT> y <!ATTLIST>.
  • Tipos de datos limitados: Solo #PCDATA (texto) o elementos anidados.
  • No soporta namespaces (espacios de nombres XML).
  • Definición inline (dentro del XML) o externa (archivo .dtd).

2. Ejemplo de XSD (XML Schema Definition)

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="biblioteca">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="libro" maxOccurs="unbounded">
          <xs:complexType>
            <xs:sequence>
              <xs:element name="titulo" type="xs:string"/>
              <xs:element name="autor" type="xs:string"/>
              <xs:element name="año" type="xs:integer"/>
              <xs:element name="precio" type="xs:decimal"/>
            </xs:sequence>
            <xs:attribute name="idioma" type="xs:string" use="optional"/>
          </xs:complexType>
        </xs:element>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>

Puntos clave de XSD:

  • Sintaxis XML: Se escribe en formato XML (a diferencia de DTD).
  • Tipos de datos avanzados: string, integer, decimal, date, etc.
  • Soporta namespaces y validación de estructuras complejas.
  • Restricciones flexibles: minOccurs, maxOccurs, pattern, etc.

Diferencias entre DTD y XSD

Característica DTD XSD
Formato Sintaxis propia (no XML) XML (esquema válido)
Tipos de datos Solo #PCDATA Strings, números, fechas, etc.
Namespaces No soportados Soporte completo
Extensibilidad Limitada Alta (herencia, grupos, etc.)
Validación avanzada No Sí (rangos, patrones, etc.)
Uso moderno En desuso (legacy) Estándar actual

DTD se usa principalmente en proyectos legacy o sistemas simples, mientras que XSD es el estándar actual para sistemas que requieren validación estricta de datos XML.

XPATH y XQUERY

Ejemplo de XPath


        //libro[precio > 30 and año > 2000]/titulo/text()

Este ejemplo de XPath:

  • Busca todos los elementos libro donde el precio sea mayor que 30 y el año sea posterior a 2000
  • Luego selecciona el texto de sus elementos titulo

Ejemplo de XQuery


        for $libro in doc("biblioteca.xml")//libro
        where $libro/precio > 30 and $libro/año > 2000
        order by $libro/titulo
        return <libro-caro>{$libro/titulo/text()}</libro-caro>

Este ejemplo de XQuery:

  • Busca los mismos libros que el ejemplo XPath
  • Los ordena por título
  • Devuelve los títulos dentro de un nuevo elemento libro-caro

Diferencias principales

Característica XPath XQuery
Propósito Navegación y selección Consulta y transformación
Complejidad Más simple Más completo y complejo
Capacidad de consulta Solo selección Selección, filtrado, ordenación, construcción de nuevos nodos
Estructura Expresión única Puede contener múltiples expresiones con lógica compleja
Resultados Retorna nodos o valores Puede retornar nodos, valores o documentos XML completos
Clausulas No tiene cláusulas Tiene cláusulas como FOR, LET, WHERE, ORDER BY, RETURN
Uso típico Dentro de otros lenguajes Como lenguaje independiente

XPath es esencialmente un subconjunto de XQuery - cualquier expresión XPath válida también es válida en XQuery. XQuery añade capacidades de consulta más potentes, incluyendo construcciones FLWOR (FOR-LET-WHERE-ORDER BY-RETURN) y la capacidad de crear nuevos documentos XML.
